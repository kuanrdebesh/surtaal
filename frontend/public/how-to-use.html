Quick Start

Open the app, load one song, verify playback, then process.

Surtaal works best when you preview the uploaded track first, then run one tool at a time and review the result before moving on.

1. Start Surtaal Run ./start.sh from the project folder, then open http://localhost:3000.
2. Pick a tool from the left menu Stem Separator, Vocal Remover, Pitch Shift, BPM & Tempo, or Audio Workshop.
3. Upload a file and use the preview bar Every upload area now includes an audio player so you can verify the correct song before processing.
4. Start processing and keep working Long jobs run in the background. You can switch tabs while progress continues.
Recommended Workflow

A practical flow for most users.

For karaoke practice Start with Vocal Remover or extract only Backing in Stem Separator, then send the result to the Workshop.
For sur adjustment Run Pitch Shift after you confirm the uploaded song and know the target key.
For slow riyaaz Use BPM & Tempo to detect the track speed first, then slow it down in small steps.
For custom live arrangements Extract only the stems you need, add them to the Workshop, then trim, mute, align, and export.

The Workshop has two important states: the empty launch view, and the loaded arrangement view where checked tracks drive the main Play button, the left panel can be resized, and track rows can be made taller or shorter together.

Tools

What each section is for.

Stem Separator

  • Select only the stems you want to extract.
  • Already extracted stems stay in the panel and are disabled from re-selection.
  • Use the shared checkboxes to play multiple stems together or send them to the Workshop.

Vocal Remover

  • Best for creating a karaoke-friendly backing track quickly.
  • Fast is the default. Best Quality is slower but cleaner on difficult mixes.

Pitch Shift

  • Use this when the song is too high or too low for your natural sur.
  • The tool keeps the detected Original Key visible while the New Key updates from your current settings.
  • Positive values raise key. Negative values lower key.

BPM & Tempo

  • Detect BPM first if you are unsure of the original speed.
  • The tool keeps Original BPM visible while New BPM updates live from the tempo factor.
  • Then adjust tempo up or down while keeping pitch stable.

Audio Workshop

  • Add stems or processed files to tracks.
  • Checked tracks are the ones the main Play button will play.
  • Resize the left track-details panel horizontally, and use the shared Height control to resize all track rows together.
  • The left panel follows the same vertical scroll as the waveform area, so the two sides stay aligned.
  • Use Clear All when you want to reset the workshop in one step.
  • Use the workshop for layering, trimming, fading, and export.

Background Jobs

  • Long operations keep a visible progress card across tabs.
  • You do not need to remain on the same screen while waiting.
Stem Separator

How to get the stems you actually need.

Pick exact stems, not just a mode If you only want Guitar or Drums, choose just that stem. Surtaal will select the smallest extraction family possible behind the scenes.
Fast vs Best Quality Fast is the right default for trial runs. Use Best Quality when artifacts matter more than speed.
Why some stems take longer Vocals and backing are usually easiest. Guitar and piano are harder and may require the 6-source model.
Use the extracted stems shelf Each new extraction is added to the panel below. You can preview it, download it, play selected stems together, or send selected ones to the Workshop.
Downloads are easier to recognize now Exported results use readable names such as Song_guitar.mp3 or Song_150_BPM.mp3 instead of random UUID-style filenames.
If a stem reaches 99% and waits, separation may already be done while Surtaal is still writing files and encoding MP3 output.
Audio Workshop

Use the Workshop as your arranging table.

  • Add stems from the separator, or add normal audio files directly.
  • The first real track load auto-fits the timeline. Later track additions keep your current zoom.
  • Each loaded track is checked for playback by default.
  • If any track checkboxes are checked, the main Play button plays only those checked tracks.
  • If no track checkboxes are checked, Play plays all tracks.
  • The left panel can be resized horizontally, and the shared Height control changes all track rows together.
  • Workshop track controls keep the detected Original key visible while the Current or pending New value changes.
  • The waveform side is the main vertical scroller, and the left panel follows it so both sides stay aligned.
  • Use volume, mute, solo, fades, and track replacement to shape the arrangement.
  • Clear All removes every track after confirmation and resets the workshop view.
  • When satisfied, use export to render the active mix.
Troubleshooting

Common issues and what to check first.

Nothing happens after clicking a button Make sure the backend is running. Restart Surtaal with ./start.sh.
Processing feels slow Stem separation is local and CPU-heavy on many Macs. Let it run in the background while you continue using the app.
Audio is not playing in the browser Chrome is the safest choice. Safari can be unreliable for Web Audio playback.
Progress moves strangely Different models report progress differently. The final stage may still include MP3 writing and cleanup after model inference ends.
Generated audio will not scrub correctly Update to the latest Surtaal build. Newer versions serve output files with byte-range support so the native audio seek bar works on generated stems and processed tracks.
Good Habits

Small habits that make Surtaal smoother.

  • Always preview the uploaded track before processing.
  • Start with Fast if you are experimenting.
  • Extract only the stems you really need.
  • Send promising stems to the Workshop instead of downloading everything first.
  • Keep Chrome open if you are doing lots of Workshop playback.
